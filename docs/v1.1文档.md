## 一、项目介绍

### 1.1 项目概述

RoomMind 是一个基于 Spring Boot + Vue 的自习室预约管理系统，旨在为高校或培训机构提供便捷的自习室座位预约服务。

### 1.2 功能特性

| 模块 | 功能 |
| :--- | :--- |
| 用户管理 | 用户注册、登录、个人信息管理 |
| 自习室管理 | 自习室列表展示、详情查看 |
| 座位预约 | 座位选择、时间段预约、预约记录管理 |
| 积分系统 | 积分获取、积分消费、积分记录 |
| 评论系统 | 预约评价、评分 |
| 轮播图管理 | 首页轮播图展示 |

### 1.3 技术栈

**后端技术栈**：
- Java 17
- Spring Boot 3.x
- MyBatis-Plus
- JWT 认证
- MySQL 8.0
- Redis 7.x
- RabbitMQ 3.x
- Maven

**前端技术栈**：
- Vue 2.x
- Element UI
- Vue Router
- Axios
- Vue CLI

**基础设施**：
- Docker / Docker Compose
- Nginx（反向代理）

## 二、本地开发指南

### 2.1 本地开发环境要求

| 软件 | 版本 | 说明 |
| :--- | :--- | :--- |
| JDK | 17 | Spring Boot 3.x 必须 |
| Maven | 3.9+ | 后端构建工具 |
| Node.js | 18+ | 前端开发环境 |
| npm/yarn | 配套版本 | 前端包管理工具 |

> **注意**：本地开发不需要安装 MySQL、Redis、RabbitMQ，这些中间件都部署在服务器的 Docker 容器中，本地通过公网 IP 连接。

### 2.2 环境验证

```bash
# 验证JDK
java -version
# 预期输出：openjdk version "17.x.x"

# 验证Maven
mvn -v
# 预期输出：Apache Maven 3.9.x

# 验证Node.js
node -v
# 预期输出：v18.x.x

# 验证npm
npm -v
# 预期输出：9.x.x 或更高
```

### 2.3 启动本地开发环境

#### 2.3.1 启动后端（开发环境）

```bash
# 进入后端目录
cd backend

# 使用dev配置启动（连接服务器中间件）
mvn spring-boot:run "-Dspring.profiles.active=dev"
```

**后端配置说明**（application-dev.yml）：
- 数据库连接：服务器 MySQL（47.94.202.210:3306）
- Redis连接：服务器 Redis（47.94.202.210:6379）
- RabbitMQ连接：服务器 RabbitMQ（47.94.202.210:5672）
- 上传路径：`./uploads-dev`（项目根目录下，不打包进jar）
- 服务器端口：`8080`

#### 2.3.2 启动前端（开发环境）

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次执行）
npm install

# 启动开发服务器
npm run serve
```

**前端配置说明**（.env.development）：
- API地址：`/api`（通过开发服务器代理转发到后端）
- WebSocket地址：`ws://localhost:8080/ws`

**vue.config.js 代理配置**：
```javascript
proxy: {
    '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        pathRewrite: { '^/api': '' },
    },
}
```

#### 2.3.3 访问地址

| 服务 | 地址 | 说明 |
| :--- | :--- | :--- |
| 前端页面 | http://localhost:5173/ | Vue开发服务器 |
| 后端API | http://localhost:8080/ | Spring Boot后端 |
| 上传图片 | http://localhost:8080/uploads/ | 静态资源映射 |

### 2.4 本地开发流程

```
1. 启动后端
   cd backend
   mvn spring-boot:run "-Dspring.profiles.active=dev"

2. 启动前端（新终端）
   cd frontend
   npm run serve

3. 浏览器访问
   http://localhost:5173/

4. 代码修改后
   - 后端：Spring Boot DevTools自动热部署
   - 前端：Vue开发服务器自动热更新

5. 测试上传功能
   - 图片保存到：项目根目录/uploads-dev/
   - 访问地址：http://localhost:8080/uploads/{userId}/{time}/{filename}
```

### 3.2 服务器中间件容器

```bash
# 启动开发中间件
cd /root/roommind-dev
```

**包含服务**：

### 3.3 服务器业务容器

#### 3.3.1 测试业务容器（roommind-dev.yml）

```bash
# 启动测试业务容器
cd /root/roommind-dev
docker-compose -f roommind-dev.yml up -d

# 查看日志
docker logs roommind-dev -f

# 重启容器
docker-compose -f roommind-dev.yml restart
```

**配置说明**：
- 使用 `application-test.yml` 配置
- 端口映射：`127.0.0.1:8080:8080`（仅本地访问）
- 上传路径：`/roommind/upload`（挂载到宿主机）
- Nginx代理路径：`/test-api/` 和 `/test-uploads/`

#### 3.3.2 生产业务容器（roommind-prod.yml）

```bash
# 启动生产业务容器
cd /root/roommind-prod
docker-compose -f roommind-prod.yml up -d
```

**配置说明**：
- 使用 `application-prod.yml` 配置
- 端口映射：`127.0.0.1:8090:8080`（仅本地访问）
- Nginx代理路径：`/api/` 和 `/uploads/`

### 3.4 Nginx配置（roommind.conf）

```nginx
# HTTP 80端口重定向到HTTPS
server {
    listen 80;
    server_name smallv.fun www.smallv.fun;
    return 301 https://$host$request_uri;
}

# HTTPS 443端口
server {
    listen 443 ssl http2;
    server_name smallv.fun www.smallv.fun;

    ssl_certificate /etc/nginx/ssl/www.smallv.fun.pem;
    ssl_certificate_key /etc/nginx/ssl/www.smallv.fun.key;

    # 前端静态资源
    location / {
        root /var/www/roommind;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 生产环境API代理
    location /api/ {
        proxy_pass http://127.0.0.1:8090/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 测试环境API代理
    location /test-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 生产环境上传文件
    location /uploads/ {
        alias /root/roommind-prod/roommind-prod-upload/;
        expires 30d;
        add_header Cache-Control public;
    }

    # 测试环境上传文件
    location /test-uploads/ {
        alias /root/roommind-dev/roommind-dev-upload/;
        expires 30d;
        add_header Cache-Control public;
    }

    # 生产环境WebSocket
    location /ws {
        proxy_pass http://127.0.0.1:8090/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 测试环境WebSocket
    location /test-ws {
        proxy_pass http://127.0.0.1:8080/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

**Nginx命令**：
```bash
# 检查配置
nginx -t

# 重载配置
nginx -s reload

# 查看状态
systemctl status nginx
```

### 3.5 环境切换说明

| 环境 | 启动方式 | API路径 | 上传路径 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 本地开发 | `mvn spring-boot:run -Dspring.profiles.active=dev` | http://localhost:8080/ | ./uploads-dev/ | 连接服务器中间件 |
| 服务器测试 | `docker-compose -f roommind-dev.yml up -d` | https://smallv.fun/test-api/ | /root/roommind-dev/roommind-dev-upload/ | 使用测试中间件容器 |
| 服务器生产 | `docker-compose -f roommind-prod.yml up -d` | https://smallv.fun/api/ | /root/roommind-prod/roommind-prod-upload/ | 使用生产中间件容器 |

> **注意**：测试环境和生产环境不能同时启动，因为它们使用不同的端口（8080/8090），但共享同一台服务器资源。

## 四、打包部署流程

### 4.1 后端打包

```bash
cd backend

# 清理旧包
mvn clean

# 打包（跳过测试）
mvn package -DskipTests

# 产物位置
# target/web-0.0.1-SNAPSHOT.jar
```

### 4.2 前端打包

```bash
cd frontend

# 生产打包
npm run build

# 产物位置
# dist/ 目录
```

### 4.3 上传服务器

```bash
# 上传后端jar包（测试环境）
scp backend/target/web-0.0.1-SNAPSHOT.jar root@服务器IP:/root/roommind-dev/roommind.jar

# 上传后端jar包（生产环境）
scp backend/target/web-0.0.1-SNAPSHOT.jar root@服务器IP:/root/roommind-prod/roommind.jar

# 上传前端静态资源
scp -r frontend/dist/* root@服务器IP:/var/www/roommind/
```
