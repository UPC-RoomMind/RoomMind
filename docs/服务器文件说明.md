## 二、文件详细说明

### 2.1 测试环境业务容器（roommind-dev）

| 文件/目录 | 路径 | 说明 |
|----------|------|------|
| roommind-dev-logs/ | `/root/roommind-dev/roommind-dev-logs/` | 测试环境后端日志目录，容器启动后日志文件存放在这里 |
| roommind-dev-upload/ | `/root/roommind-dev/roommind-dev-upload/` | 测试环境上传文件目录，用户上传的图片存放在这里，Nginx映射 `/test-uploads/` |
| roommind.jar | `/root/roommind-dev/roommind.jar` | 测试环境后端代码打包后的jar包，挂载到容器内 `/roommind/roommind.jar` |
| Dockerfile | `/root/roommind-dev/Dockerfile` | 测试环境Docker镜像构建文件，基于openjdk:17-jdk-slim，启动命令使用 `--spring.profiles.active=test` |
| roommind-dev.yml | `/root/roommind-dev/roommind-dev.yml` | 测试环境docker-compose配置文件 |

**roommind-dev.yml 关键配置**：
```yaml
services:
  roommind-dev:
    image: roommind:dev              # 镜像名称
    container_name: roommind-dev     # 容器名称
    ports:
      - "127.0.0.1:8080:8080"       # 宿主机端口8080映射到容器端口8080，仅本地访问
    volumes:
      - ./roommind.jar:/roommind/roommind.jar           # jar包挂载
      - ./roommind-dev-logs:/roommind/logs              # 日志目录挂载
      - ./roommind-dev-upload:/roommind/upload          # 上传文件目录挂载
    networks:
      - dev_net                      # 加入开发环境网络，与中间件互通
    restart: always                  # 容器退出后自动重启
```

## 六、部署流程

### 6.1 首次部署

```bash
# 1. 安装Docker和Docker Compose
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 2. 创建目录结构
mkdir -p /root/roommind-dev/roommind-dev-logs
mkdir -p /root/roommind-dev/roommind-dev-upload
mkdir -p /root/roommind-prod/roommind-prod-logs
mkdir -p /root/roommind-prod/roommind-prod-upload
mkdir -p /etc/nginx/ssl

# 3. 上传配置文件
scp middleware-dev.yml root@47.94.202.210:/root/
scp middleware-prod.yml root@47.94.202.210:/root/
scp roommind-dev/* root@47.94.202.210:/root/roommind-dev/
scp roommind-prod/* root@47.94.202.210:/root/roommind-prod/
scp nginx.conf root@47.94.202.210:/etc/nginx/conf.d/roommind.conf
scp www.smallv.fun.pem root@47.94.202.210:/etc/nginx/ssl/
scp www.smallv.fun.key root@47.94.202.210:/etc/nginx/ssl/

# 4. 启动中间件
cd /root && docker-compose -f middleware-dev.yml up -d
cd /root && docker-compose -f middleware-prod.yml up -d

# 5. 构建并启动业务容器
cd /root/roommind-dev && docker build -t roommind:dev . && docker-compose up -d
cd /root/roommind-prod && docker build -t roommind:prod . && docker-compose up -d

# 6. 加载Nginx配置
nginx -t && nginx -s reload
```

### 6.2 更新代码

```bash
# 1. 停止容器
cd /root/roommind-dev && docker-compose down
cd /root/roommind-prod && docker-compose down

# 2. 上传新的jar包
scp roommind.jar root@47.94.202.210:/root/roommind-dev/
scp roommind.jar root@47.94.202.210:/root/roommind-prod/

# 3. 重新启动容器
cd /root/roommind-dev && docker-compose up -d
cd /root/roommind-prod && docker-compose up -d
```

### 6.3 更新前端

```bash
# 1. 上传dist文件
scp -r dist/* root@47.94.202.210:/var/www/roommind/

# 2. 刷新Nginx缓存
nginx -s reload
```

## 七、常用操作命令

### 7.1 容器管理

```bash
# 查看运行中的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a

# 查看容器日志
docker logs -f roommind-dev
docker logs -f roommind-prod

# 进入容器
docker exec -it roommind-dev bash

# 停止容器
docker stop roommind-dev

# 删除容器
docker rm roommind-dev

# 重新构建镜像并启动
cd /root/roommind-dev && docker build -t roommind:dev . && docker-compose up -d
```

### 7.2 中间件管理

```bash
# 启动开发环境中间件
cd /root && docker-compose -f middleware-dev.yml up -d

# 启动生产环境中间件
cd /root && docker-compose -f middleware-prod.yml up -d

# 查看中间件日志
docker logs -f mysql-dev
docker logs -f redis-dev
docker logs -f rabbitmq-dev
```

### 7.3 Nginx管理

```bash
# 验证配置
nginx -t

# 重新加载配置
nginx -s reload

# 查看Nginx状态
systemctl status nginx

# 查看访问日志
tail -f /var/log/nginx/access.log

# 查看错误日志
tail -f /var/log/nginx/error.log
```

## 八、注意事项

1. **测试环境和生产环境使用不同的网络**：`dev_net` 和 `prod_net`，中间件完全隔离
2. **生产环境端口仅本地访问**：MySQL、Redis、RabbitMQ只绑定 `127.0.0.1`，不对外暴露
3. **上传文件目录已挂载**：容器内的 `/roommind/upload` 挂载到宿主机，容器删除后文件不会丢失
4. **日志目录已挂载**：容器内的 `/roommind/logs` 挂载到宿主机，方便查看日志
5. **容器自动重启**：配置了 `restart: always`，服务器重启后容器会自动启动
6. **资源限制**：每个容器都配置了CPU和内存限制，防止资源耗尽
